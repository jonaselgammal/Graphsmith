# Open Standard Notes

The eventual standard should define:
- package format
- graph model
- primitive op vocabulary
- type vocabulary
- effect vocabulary
- registry API
- trace format
- promotion candidate format

## Design principles

- portable over vendors
- deterministic execution semantics
- human-readable packaging
- machine-friendly structure
- versioned and extensible
- supports local-first development

## Standardization advice

Do not freeze the standard too early.
First build a useful reference runtime and gather a small library of real skills.
Then stabilize:
- package format
- registry API
- primitive op semantics

# Sprint 80: Graph-Native Skill Synthesis v1

This sprint adds the first bounded graph-native synthesis path.

Before this change:

- closed-loop generation mainly produced single-step deterministic skills
- larger fallback graphs were reusable only implicitly by rebuilding them again

After this change:

- selected structural fallback graphs can be synthesized into real reusable
  skill packages
- the synthesized package gets:
  - a stable generated skill id
  - a normal Graphsmith package layout (`skill.yaml`, `graph.yaml`,
    `examples.yaml`)
  - typed inputs and outputs inherited from the composed graph contract
  - a deterministic smoke example generated by executing the package
- closed-loop then swaps the inline fallback graph for a single `skill.invoke`
  of the synthesized subgraph skill

Current bounded scope:

- synthesized package creation is enabled for structural fallback families:
  - structured numeric composition
  - branch-with-different-body fallback
- simple inline pipelines are not broadly converted yet
- smoke examples use the echo/stub-style local provider path and are intended as
  deterministic validation artifacts, not semantic quality guarantees

Why this matters:

- Graphsmith can now start turning solved composed regions into first-class
  reusable skills rather than treating them as throwaway plans.
- This is the first concrete step from “templated op generation” toward
  “generated subprograms with contracts and promotion potential.”
